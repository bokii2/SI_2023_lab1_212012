Со командата mkdir се креира нов директориум, а со командата cd се пристапува кон него.
Со git init се иницијализира Git во директориоумот.
Со git config --global се поставуваат конфигурациски вредности на глобален или локален проект. А со додавање на user.name <име> и user.email <маил> се поставуваат името и маилот на корисникот. 
Со git remote add origin <линк од remote repo на Github> се поврзува со оддалечениот репозиториум.
touch e команда за креирање нов документ, додека пак nano е команда за креирање и отварање на нов документ или отварање на веќе постоечки.
После креирање и модифицирање на документ, се пристапува со командата git add <name> за документот се однесе во stage фаза.
git commit e команда со која се прави нов комит, додека пак со -m се пишува коментар за комитот.
git push -u origin <име на гранка од локален репозиториум> e команда со која се синхронизира remote repo-то на Github.
Со git checkout -b <име> се прави нова гранка на локалното репо и се префрла на таа нова гранка, додека пак git checkout <име> е команда само за префрлување на веќе постоечка гранка.
Промените во документот се прават со nano командата и после тоа со командата git commit се прават бараните комити.
Со git merge <име на гранка> се спојуваат гранките со master гранката, но предходно треба да се префрлиме на master гранката.
Со git push --all се синхронизираат сите промени кон remote repo-то, a --force е за присилно синхронизирање.
Со git log се прикажува кој ги направил комитите и кога.
